# Nautical Compass — Helm System Spec

## Purpose
The **Helm** is the main command interface for Nautical Compass.

It is not a generic dashboard.  
It is a **unified system-status instrument panel** that can represent:

- legal matters
- gig worker positioning
- business growth
- labor coordination
- dispatch operations
- vessel operations later

The core rule is simple:

**Everything in NC is treated as a system in motion.**  
The Helm shows whether that system is stable, under pressure, off course, exposed, or advancing.

## Product Role
The Helm gives the user an instant answer to:
- where they are now
- what is under control
- what needs attention
- what direction they are moving
- what to open next

## Core Instruments
Use 6 primary instruments and 1 alert state:

1. **System Load**
2. **Flow**
3. **Risk Index**
4. **Signal**
5. **Direction**
6. **Velocity**
7. **Global Alert Ring**

## Universal Rule
The same instruments must work across:
- gig workers
- businesses
- legal matters
- dispatch operations
- marine mode later

## Homepage Rule
Do not clutter the homepage.  
Do not add more widgets.  
Make the instruments smarter.

## Public Mode
Show demo-state intelligence only:
- System Load: Moderate
- Flow: Building
- Risk Index: Elevated
- Signal: Active
- Direction: Advancing
- Velocity: Moving

## Logged-in Mode
Show actual user state from scoring and intake data.

## Action Logic
Each gauge should open a focused next step:
- Flow → invoices / receivables
- Risk → structure / protection
- Signal → leads / dispatch
- Direction → roadmap / upgrade
- Velocity → finish setup / resume intake

## Final Position
The Helm is the universal operating language of Nautical Compass.

It should tell the user at a glance:
- what condition they are in
- what is strong
- what is exposed
- where to go next
